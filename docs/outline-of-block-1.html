  3. The modeling process
    1. NTI:
      1. Examples of mechanistic model situations:
        1. Heat engine: Work extracted from heat when going through a complete cycle.
          1. Need to know that pressure times volume is work.
          2. Need to know how a heat engine works: compression (work in), heating, expansion (work out), cooling
          3. Area of loop is work
        2. Law of mass action: speed of reaction as a function of the concentration of the components. Should it be the product, the sum, … what?
      2. Examples of data modeling
        1. Effectiveness of vaccines.
          1. Measure age, health, level of exposure, vaccination status, whether and when fell ill with COVID.
          2. Model probability versus time as a function of age, health, exposure, vaccination. That function can then be compared for vaccination and non-vaccination status. If p(t | vaccinated) < 0.1 p(t | not vaccinated), holding other variables constant, then vaccine is more than 90% effective.
    2. Topics
      1. Exponentials
        1. function asymptotically approach baseline, but since it takes an infinite time to get there, we start by making an educated guess. b. If there’s relevant theory, use that.
          1. E.g. objects cool to ambient temperature is a nice theory. But even that isn’t complete: radiative balance with environment, latent heat of evaporation, …
          2. Things with a zero baseline: loss of velocity with friction or air resistence over time; value of invested money; discounted value of future money.
        2. Once you make a baseline guess, check it. For instance, for exponential decay …
          1. Figure out how long it takes to go from distance \(h\) from baseline to distance \(h/2\). Then measure how long it takes to go from distance \(2h\) to \(h\). If the baseline is correct, those two durations should be the same. If the \(h \longrightarrow h/2\) duration is shorter than \(2h\longrightarrow h\) duration, then baseline was too high. Work through an example and iterate in on reasonable answer.
          2. Given baseline, finding doubling or halving time and convert this into the value of \(k\) in \(e^{kt}\).
      2. Sinusoids.
        1. Easy to see parameters: baseline, amplitude for sinusoid, first positive-going baseline crossing.
        2. Hard to see parameter: period. Best to measure duration to go through several cycles, then divide by number of cycles to get period.
      3. Hump/gaussian function. a. Center is pretty easy: argmax b. Emphasize rules of thumb for spotting the width:
        1. half-width at 60% height.
        2. distance from center marking transition from concave down to concave up. c. This width has a strange name: “standard deviation” d. German 10 mark note commemorates Gauss and gaussian
      4. Sigmoid a. Center is argument that gives rise to 1/2 of asymptotic value. b. Width is distance from center to point where function value is 84% of full range or 16% of full range.
    3. Readings: Chapter ??
  4. Slope function
    1. NTI:
      1. How do broaden the idea of describing the slope of a line to a general function?
      2. No \(h\) at this point
      3. The slope function is a function \(s()\) which is built from a function \(f()\)
    2. Topics
      1. Average rate of change. (Example: Quarterly return of stocks. HW of tree harvesting.) (Example 2: Sine wave with difference around 1 period. Or maybe this becomes low-pass filter example in accumulation.)
      2. \(s(x) \equiv \frac{f(x + 0.1) - f(x)}{0.1}\)
      3. Except for linear function, slope function depends on how big \(h\) is. So let’s fix \(h\) while we figure things out.
      4. Every function has a slope function.
      5. It’s helpful to name functions so we know where they come from. We’re going to use \(f'()\) to stand for the slope function of \(f()\)
    3. Reading: Chapter ??
